### Henning · building BeastForge

> **AI agents write code fast — and forget *why*.**
> Close the chat and the plan, the decisions, the dead ends are gone.

I work on **Verifiable Agentic Engineering** — running AI-assisted projects so the
memory lives in the **repo, not the chat**. The plan, the live state, every decision,
the open questions sit in version-controlled files the agent re-reads each session.

- 🧠 **Memory in the repo** — `STATE` · `DECISIONS` · `MASTERPLAN` · open questions
- 🔁 **Resume cold** — the agent picks up where it left off, no re-explaining
- ✅ **You stay the auditor** — every change traces to a decision; reproduce it *without the AI*

> ### Because you can't audit a vibe.

**BeastForge** is the open template that puts it into practice — spawn a project
(*a "beast"*) and it keeps its own memory, decisions, and reproducible trail as you
work. *Public release coming.*
